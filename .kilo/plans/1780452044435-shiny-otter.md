# Deep System Analysis & Corrected Hardening Plan — defi-arbitrage-core-public-mvp

> **Corrected plan based on exhaustive file-by-file investigation.** Previous plans contained several inaccuracies that have been verified and corrected against actual source code.

## Phase 1: Verified System Understanding

### Architecture (confirmed from source)

This is a **deterministic, offline DeFi arbitrage analysis kernel**:

- **Language**: Python 3.10+ (canonical, zero hard runtime deps), optional Rust 2021 (process-isolated accelerator)
- **Data flow**: Local JSON snapshot → `JsonFileProvider` → `validate_snapshot_payload()` + dataclass `validate()` → `AnalysisEngine` bounded DFS → policy filter → `JsonReporter`/`TextReporter`
- **Trust boundary**: Local untrusted input only; no network, no keys, no signing, no execution
- **Dual package**: `defi_arbitrage_core` (public name) re-exports `arbcore` (internal name); both CLI entrypoints map to same code

### Key Components (confirmed)

| Module | Role | Key Lines |
|--------|------|-----------|
| `models.py` | Frozen dataclasses: `Edge`, `MarketSnapshot`, `RiskPolicy`, `Opportunity` | Edge.validate():71-99 already has MAX_METADATA_BYTES/KEYS/DEPTH; RiskPolicy.validate():158-174 already has `max_notional >= min_liquidity` at line 169 |
| `validation.py` | Dependency-free payload shape checks | `validate_snapshot_payload()` rejects unknown keys, enforces sizes |
| `providers.py` | `JsonFileProvider` — reads local file, calls `validate_snapshot_payload()` then `MarketSnapshot.validate()` | No path restriction, no symlink check, no file-size limit |
| `engine.py` | Bounded DFS cycle search | `_walk()` at line 37: recursive, depth-limited by `max_hops` (≤8), has `math.isfinite()` guard at line 52, `MAX_EDGE_DEGREE=500` check at line 26 |
| `service.py` | Rust subprocess adapter | `_resolve_binary()`: resolves path but no allowlist; `available()` checks `Path.exists()` then `shutil.which()`; `preexec_fn` sets `RLIMIT_AS` on POSIX only, silently ignores failures |
| `contracts/validation.py` | Static Solidity/Vyper template validator | Checks secrets, dangerous primitives, hashes, paths under `contracts/` |
| `diagnostics.py` | Operational diagnostics | Emits `str(Path.cwd())` (absolute path), `sys.executable` (full path), resolved policy values |

### Invariants (confirmed)

1. **Fail-closed**: Unknown keys rejected; malformed input raises `SnapshotError`/`PolicyError`
2. **Bounded analysis**: `max_hops` caps DFS depth (≤8); `max_results` caps output (≤1000); `MAX_EDGE_DEGREE=500` caps node degree; Python already has `math.isfinite()` guard
3. **Rust is optional**: Python is canonical; Rust fallback is automatic via `analyze_with_optional_rust()`
4. **No secrets in scope**: `.env.example` documents non-secret config only
5. **Deterministic**: Same input → same output within float precision

### New findings NOT captured in prior plans

1. **`engine.py:_walk` uses `visited_assets = {hop.source for hop in next_route}`** at line 65 — a new set is allocated on every recursive call. With `max_hops=8` this is fine, but it's O(branches^hops * path_len) memory allocation overhead.
2. **Rust `walk` has same set allocation** at `main.rs:349`: `let visited: BTreeSet<String> = next_route.iter().map(...).collect()` — same pattern.
3. **`_opportunity_from_dict` does NOT recompute profit_bps from gross_return** — it trusts the Rust binary's `profit_bps` field. A buggy or compromised Rust binary could return inconsistent numerics that pass validation. Confirmed at `service.py:171-177`.
4. **The `canonical_key` implementations differ in type but should produce equivalent results**: Python uses `tuple` + `min()` on tuples; Rust uses `Vec<String>` + lexicographic comparison. Both should be equivalent for ASCII asset symbols, but Unicode normalization is not tested.
5. **`validate_rust_service.py` hardcodes `target/debug/`** at line 13 — will fail silently if binary is compiled with `--release`.
6. **`fixtures/parity/timestamp_validation.json` documents sub-second and `+00:00` timestamps as valid** (lines 5-7), but both engines reject them. This is a real documentation-vs-reality gap.
7. **`secret_scan.sh` uses `grep -nEI`** which relies on extended regex — the pattern includes `ghp_`, `github_pat_`, etc. but does not scan binary files or git history. Confirmed at `scripts/secret_scan.sh:22`.
8. **`config.py:_read_float` and `_read_int`** return defaults for empty string AND `None` — but `os.getenv()` returns `None` for unset vars and `""` for empty-string vars. Both are handled identically. This is correct behavior but worth noting.

## Phase 4: Corrected Remediation Plan

### Sequence Priority (corrected based on verified findings)

1. **Critical** (C-1 through C-3): Path traversal, symlink following, algorithmic DoS via unbounded call count
2. **High** (H-1 through H-7): Binary allowlist, memory exhaustion, Rust parity gaps
3. **Medium** (M-1 through M-8): Diagnostics leakage, CLI silent fallback, schema drift
4. **Low** (L-1 through L-5): Docker hardening, CI pinning, metadata sanitization

### Detailed Remediation Actions

#### CRITICAL

**C-1: Path Traversal / No Path Restriction**
- **Fix**: Add `_resolve_safe_path(path, allowed_root)` helper that: (1) resolves to absolute path via `resolve()`, (2) rejects symlinks (`is_symlink()` check on path and all parents), (3) verifies resolved path is within `allowed_root` via `is_relative_to()`. Apply to `JsonFileProvider.__init__()`, `Settings.from_env()` for `ARBCORE_PROVIDER_FILE`, `validate_contract_workspace()` for manifest path, and `RustAnalysisService._resolve_binary()`.
- **Tradeoff**: Restricts flexibility. Add `ARBCORE_ALLOWED_PATHS` env var (colon-separated list) with sensible defaults (repo root, `/tmp`).
- **Implementation**: New helper in `validation.py` or new `path_utils.py` module.

**C-2: Symlink Following**
- **Fix**: Covered by C-1's `_resolve_safe_path()` which rejects symlinks.
- **Tradeoff**: Same as C-1.

**C-3: Unbounded DFS Call Count**
- **Fix**: Add `self._visit_count = 0` in `AnalysisEngine.__init__`; increment in `_walk()`; raise `SnapshotError` if `_visit_count > self._max_visits`. Default `_max_visits = 1_000_000`. Also add optional `timeout_seconds` parameter using `signal.alarm` (POSIX) or `threading.Timer` (cross-platform).
- **Tradeoff**: 1M visits is generous for real snapshots but blocks exponential blowup. Make configurable via env var `ARBCORE_MAX_VISITS`.

#### HIGH

**H-1: No Binary Path Allowlist**
- **Fix**: In `RustAnalysisService.__init__()`, after `_resolve_binary()`, verify the resolved path: (1) basename matches `defi-arbitrage-core-rs*` pattern, OR (2) resolved path is within the repository's `rust/` directory. Reject with `RustServiceUnavailable` if neither.
- **Tradeoff**: Removes ability to use custom Rust binaries from arbitrary locations. Given narrow project scope, acceptable.

**H-2: Binary Path Symlink**
- **Fix**: Covered by C-1's `_resolve_safe_path()` applied to `RustAnalysisService._resolve_binary()`.

**H-3: Unbounded File-Size Read**
- **Fix**: In `JsonFileProvider.load_snapshot()`, add `if self.path.stat().st_size > MAX_SNAPSHOT_BYTES: raise SnapshotError(...)`. Set `MAX_SNAPSHOT_BYTES = 50 * 1024 * 1024` (50 MB). Define constant in `models.py` alongside other limits.
- **Tradeoff**: 50 MB is generous for any realistic snapshot (10,000 edges × ~5KB/edge = ~50MB). Make configurable if needed.

**H-4: Aggregate Metadata Memory**
- **Fix**: In `MarketSnapshot.validate()`, after per-edge checks, add: `total_metadata_bytes = sum(len(json.dumps(edge.metadata, separators=(',', ':'), default=str)) for edge in self.edges)`; raise if `> MAX_TOTAL_METADATA_BYTES` (e.g., 10 MB).
- **Tradeoff**: Adds O(n) pass over edges, but n ≤ 10,000 and this only runs once per snapshot.

**H-5: Rust Missing Safety Checks**
- **Fix**: In `rust/arbcore-rs/src/main.rs`:
  - Add `MAX_METADATA_KEYS`, `MAX_METADATA_BYTES`, `MAX_METADATA_DEPTH` checks in `normalize_edge()`.
  - Add `MAX_EDGE_DEGREE` check in `analyze()` before building graph (match Python's behavior at `engine.py:26-29`).
  - Add `MAX_SNAPSHOT_EDGES` check after parsing.
- **Tradeoff**: Rust becomes stricter; may reject snapshots that previously passed. Since Python is canonical and Rust is optional, this aligns behavior.

**H-6: Rust `partial_cmp` unwrap**
- **Fix**: Replace `.unwrap_or(Equal)` with explicit check: `if a.profit_bps.is_nan() || b.profit_bps.is_nan() { std::cmp::Ordering::Equal } else { b.profit_bps.partial_cmp(&a.profit_bps).unwrap() }`. NaN should never reach this point after validation, but this makes the behavior explicit.
- **Tradeoff**: None — this is strictly safer.

**H-7: Silent Resource Limit Failure**
- **Fix**: In `_pre_exec_resource_limit()`, log a warning when limits cannot be set: `import warnings; warnings.warn("RLIMIT_AS not set on this platform; Rust memory limit is disabled")`. Do not raise, but make silent failure observable.
- **Tradeoff**: None — additive observability.

#### MEDIUM

**M-1: Diagnostics Path Leakage + Code/Fixture Mismatch**
- **Fix**: In `collect_diagnostics()`:
  - Replace `sys.executable` with `Path(sys.executable).name` (e.g., `"python3"`).
  - Replace `str(Path.cwd())` with `"."` (always relative sentinel) or `os.path.relpath(Path.cwd())`.
- **Update**: Fix `fixtures/golden_outputs/diagnostics.json` to match new behavior.
- **Update**: `validate_golden_fixtures.py` value checks should be updated to match.
- **Investigate FIRST**: The current fixture shows `"cwd": "."` but code shows `str(Path.cwd())`. **Must determine if code was recently changed or fixture is stale before editing either.**

**M-2: Error Message Path Leakage**
- **Fix**: In error messages, use `self.path.name` instead of `self.path`. In `validation.py`, use relative paths or basenames.
- **Implementation**: Targeted string changes in `providers.py`, `validation.py`, `registry.py`.

**M-3: Silent CLI Fallback**
- **Fix**: Add explicit warning to stderr when falling back to default example: `print("warning: no snapshot specified; using default example file", file=sys.stderr)`. Alternatively, make `--snapshot` required or error when both `args.snapshot` and `ARBCORE_PROVIDER_FILE` are unset. **Decision needed**: warn vs. error.
- **Tradeoff**: Warning preserves backward compatibility; error is safer for automation.

**M-4: Wildcard Re-exports**
- **Fix**: Replace `from arbcore import *` in `defi_arbitrage_core/__init__.py` with explicit imports: `from arbcore import (AnalysisEngine, ArbCoreError, ...)` matching `arbcore.__all__`. Replace `from arbcore.providers import *` with `from arbcore.providers import JsonFileProvider, MarketDataProvider`. Same for `reporters.py`.
- **Implementation**: Straightforward; no behavior change.

**M-5: Float Precision Edge Case**
- **Fix**: Document that `profit_bps` may be slightly negative for zero-profit cycles due to float arithmetic, and that `_passes_policy` handles this via `>= min_profit_bps` comparison. No code change needed if documented.
- **Alternative**: Add epsilon: `profit_bps > -1e-9` in `is_profitable` property. But this is already handled by policy filter.

**M-6: Schema/Runtime Divergence**
- **Fix**: Add a test that introspects `Edge.validate()`, `MarketSnapshot.validate()`, and `RiskPolicy.validate()` and cross-references constraints with `schemas/*.json`. Ensure every runtime numeric bound has a corresponding schema constraint.
- **Implementation**: New test function in `tests/test_validation_schema.py` or dedicated file.

**M-7: SHA256 Case Sensitivity**
- **Fix**: Change `SHA256_RE` to `re.compile(r"^[a-fA-F0-9]{64}$")`. Normalize to lowercase in `parse_contract_manifest()`: `source_sha256=_string(item, "source_sha256", index).lower()`.
- **Implementation**: One-line regex change + `.lower()` call.
