# Defi-Arbitrage-Core: Comprehensive System Analysis & Hardening Plan

System: `defi-arbitrage-core-public-mvp` | Analyst: Senior Systems Architect & Cybersecurity Consultant | Date: 2026-06-02

---

## EXECUTIVE SUMMARY

Six confirmed gaps prevent public release: (1) `fixtures/golden_outputs/diagnostics.json` leaks an absolute path from a different internal project (`"/root/defi-arbitrage-bot"`); (2) `validate_golden_fixtures.py` checks key structure only, not values, so CI never catches it; (3) `fixtures/parity/timestamp_validation.json` documents sub-second / `+00:00` timestamps that the actual Python and Rust parsers reject; (4) Python `_walk` and Rust `walk` have no `math.isfinite()` / `is_finite()` guard on intermediate `gross_return`, risking silent `Inf` profit propagation; (5) `Edge.metadata` has no byte-depth or key-count bound; (6) `examples/custom_provider.py` imports from `defi_arbitrage_core.reporters` (wildcard re-export fragility). Additionally, (7) README.md lacks an alpha/unreleased banner despite SECURITY.md/STATUS.md blocking public release.

---

## HIGH-LEVEL ARCHITECTURE

CLI (`arbcore.cli` / `defi_arbitrage_core.cli`) → Provider (`JsonFileProvider`) → Validation (`validate_snapshot_payload`, schema checks) → Engine (`AnalysisEngine` Python / optional `RustAnalysisService`) → Reporter (`TextReporter` / `JsonReporter`). Models: `Edge`, `MarketSnapshot`, `RiskPolicy`, `Opportunity`. Errors: `ArbCoreError` hierarchy (no secrets in messages). Contracts: static-template validator (Solidity/Vyper). No network, no keys, no signing, no execution, no deployment.

| ID | Sev | Component | Issue |

## OPEN QUESTIONS / CLARIFICATIONS NEEDED

1. **Should `diagnostics.json` `cwd` be normalized in test (`validate_golden_fixtures.py`) or removed from fixture?** Current plan proposes Option B (normalize in validator) as it catches future drift. But removing from fixture (Option A) is simpler. Need maintainer decision.
2. **Is `MAX_REASONABLE_HOPS=8` a hard ceiling?** If users need deeper cycle analysis, the combinatorial explosion (P-1) becomes a real DoS vector. Consider iterative deepening with result-count early-exit.
3. **Should Vyper template be removed or completed?** Manifest references `contracts/vyper/FlashArbExecutor.vy` which exists and validates. Intentional long-term support or accidental inclusion?
4. **What is the credential rotation evidence process template?** Manual gate requires attestation but no checklist exists for *what* credentials were rotated.
